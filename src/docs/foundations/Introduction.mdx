# Hailstorm Design System

Welcome to the Hailstorm Design System documentation. This design system provides a consistent set of design tokens, components, and patterns for building Abusix products.

## Core Principles

- **Consistency** - Use semantic tokens to ensure visual consistency across all components
- **Accessibility** - Support for light and dark themes with proper contrast ratios
- **Flexibility** - Tokens are designed to be easily customizable and extensible

## Design Tokens

Design tokens are the visual design atoms of the design system. They are named entities that store visual design attributes.

## Theming

The design system supports light and dark themes. Theme switching is handled via:

- CSS class (`.dark`) on the `<html>` element
- Data attribute (`data-theme="dark"`) on the `<html>` element

Both methods are supported for maximum flexibility.

### Using the Theme

```tsx
import { ThemeProvider, useTheme } from "@abusix/hailstorm";

function App() {
    return (
        <ThemeProvider defaultTheme="system">
            <YourApp />
        </ThemeProvider>
    );
}

function ThemeToggle() {
    const { theme, setTheme } = useTheme();

    return (
        <button onClick={() => setTheme(theme === "dark" ? "light" : "dark")}>Toggle theme</button>
    );
}
```
